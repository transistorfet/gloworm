- I've renamed memory_area to memory_segment, and memory_object to memory_region to be more descriptive.  A memory segment is a
  single mapped entry in the memory_map.  A memory_region is a pre-allocated chunk of memory that is divied up between different
  segments in a memory map.  It's only used when virtual memory is not used, since otherwise it will use and allocate pages on a
  per-page basis, with a per-page refcount and per-page sharing between segments.  The memory_region struct is itself allocated, and
  holds a refcount that is used to share regions (which also means an entire region is kept around until no processes use any part
  of that region).  Since I'm still using the separate stack/heap allocation and code/data allocation instead of the shadow ram
  method, and since most processes fork and then exec right after anyways, it shouldn't actually keep much in memory
